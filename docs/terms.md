# DotDet Terms of Use

> This is not legal advice. These terms are provided for the DotDet preview
> project and may need review before broader production or commercial use.

DotDet v0.1 Preview is an early production-readiness analysis workbench for
ASP.NET Core solutions. By using DotDet, you agree to use it responsibly and to
review its output with engineering judgment.

## Preview status

DotDet is a preview project under active development and calibration. Features,
rules, scores, reports, GitHub workflows, storage behavior, and availability may
change without notice.

DotDet does not guarantee that a repository is production ready, secure,
compliant, or free of defects. Reports may include false positives, false
negatives, incomplete findings, or recommendations that do not fit your
architecture or deployment environment.

## Permitted use

You may use DotDet to analyze ASP.NET Core and .NET repositories that you are
authorized to access and evaluate. You are responsible for making sure you have
the right to submit uploaded ZIP files, public repositories, private
repositories, source files, configuration, metadata, and related materials to
DotDet.

You may use generated reports as engineering review material, pull request
context, planning input, or internal documentation, but you remain responsible
for validating the findings and deciding whether a change is appropriate.

## Prohibited use

Do not use DotDet to:

- Analyze repositories or source code you do not have permission to access.
- Attempt to access another user's repositories, reports, tokens, history, or
  suppressions.
- Abuse, scrape, overload, automate excessive requests, or bypass rate limits.
- Upload malicious archives, attempt path traversal, or probe the service host.
- Use generated reports to claim certification, audit completion, legal
  compliance, or security approval that DotDet has not provided.

## User responsibility

DotDet is not a substitute for manual code review, security testing, threat
modeling, dependency review, operational readiness review, legal review, or
professional judgment. You are responsible for the decisions you make based on
DotDet output.

Before acting on a finding, review the evidence, confidence, detection method,
affected files, recommended pattern, and your own application context.

## Reports and recommendations

DotDet reports are static analysis outputs. They may identify production risks,
architecture concerns, configuration issues, migration risks, dependency
injection concerns, and API readiness signals. These findings are not proof of
runtime behavior.

Scores and grades are intended to help prioritize review. They are not a release
approval, warranty, guarantee, certification, or compliance statement.
